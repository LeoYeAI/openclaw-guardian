## openclaw-backup skill 完成（2026-03-02）

### 背景
Leo 要求：定期一键保存 OpenClaw 所有配置+记忆+skills，支持还原到新实例。

### 构建内容
- `scripts/backup.sh` — 全量备份（workspace/config/skills/cron/scripts/identity）
- `scripts/restore.sh` — 还原（支持 --dry-run 预览，还原前自动保存快照）
- `scripts/schedule.sh` — 系统 cron 定时备份配置
- `references/what-gets-saved.md` — 备份内容说明文档
- 已打包：`~/.openclaw/workspace/skills/openclaw-backup.skill`
- 已 commit：git commit fcda41f

### 关键设计决策
- 备份格式：tar.gz（单文件，带时间戳，自动保留最近7个）
- **不备份**：credentials/auth tokens（安全）、日志、二进制媒体
- 还原前自动保存 pre-restore 快照
- 迁移流程：备份 → scp → restore.sh --dry-run → 执行 → 重新 pair channels

### 自动化
- 已添加 OpenClaw cron job（ID: 08d72872）：每日 UTC 03:00 自动备份
- 备份输出目录：/tmp/openclaw-backups/

### rsync 依赖
- 容器内原本没有 rsync，已 apt-get install -y rsync

## myclaw-backup 迭代记录（2026-03-02 续）

### v1.1 — 迁移后无需重新 pair
- 新增备份：credentials/（telegram-allowFrom/pairing）+ telegram/update-offset
- bot token 本来在 openclaw.json，已备份；credentials 是 pairing session 状态
- 还原后 Gateway 直接用原 bot token + credentials 恢复连接，无需操作

### v1.2 — 补全漏掉的目录（全面审计后发现）
- 补充备份：`agents/main/`（model provider config + apiKey）+ session history（24个.jsonl）
- 补充备份：`devices/`（paired.json 配对节点）
- 有意不备份：openclaw.log、canvas/index.html（系统静态）、subagents/runs.json（临时状态）
- 备份大小从 80KB 增至 2.8MB（含对话历史）

### v1.3 — 内置 HTTP 服务器
- 新增 scripts/server.js（纯 Node.js 无依赖）+ scripts/serve.sh
- Web UI：创建备份、下载 .tar.gz、上传、dry-run 预览、一键还原
- HTTP API：GET /backups、POST /backup、GET /download/:file、POST /upload、POST /restore/:file
- 用于迁移场景：旧机器 serve → 浏览器下载 → 新机器上传 → 还原

### v1.4 — 修复 ClaWHub 安全扫描问题（Leo 截图反馈）
ClaWHub OpenClaw scanner 标注 Suspicious HIGH CONFIDENCE，4个问题：
1. 依赖未声明 → SKILL.md metadata 加 requires: [node, rsync, tar, python3, openclaw]
2. HTTP server 无 token 时完全开放 → token 改为强制必填，无 token 拒绝启动
3. /health 暴露 backupDir 路径 → 改为只返回 {status:ok, service:myclaw-backup}
4. restore 无确认步骤 → CLI 加 "type 'yes'" 交互确认；HTTP API 加 confirm=1 参数要求；Web UI 改为 prompt("Type YES")
- 新增安全 headers：X-Content-Type-Options, X-Frame-Options, Cache-Control
- SKILL.md description 加 TRUST BOUNDARY 说明
