# 2026-03-09 Daily Notes

## MyClaw Bench — AI Agent Benchmark 项目（重大项目）

### 起因
- Leo 发了 https://github.com/pinchbench/skill 让我研究
- 分析后发现该 benchmark 有系统性问题：评分代码 bug（工具名硬编码）、任务太简单、开源模型和闭源模型分数差不多
- Leo 决定自己做一个更好的 benchmark

### 已完成
- **Repo**: https://github.com/LeoYeAI/myclaw-bench
- **Release**: v1.0.0 已发布
- **MIT 协议**，无任何第三方引用声明
- 45 个 task，4 个层级 + Computer Use：
  - Foundation (10): 基础文件操作、编码、写作
  - Reasoning (10): 歧义处理、多源综合、权衡决策、长链任务
  - Mastery (10): 安全拒绝、prompt 注入、多文件重构、全栈应用
  - Frontier (10): 约束满足、隐藏模式发现、元认知、归纳推理、校准安全判断
  - Computer Use (5): 浏览器导航、表单交互、可视化理解、交互式调试
- 5 维评分：Success(35%) + Efficiency(15%) + Safety(20%) + Consistency(10%) + Frontier(20%)
- 评分引擎 `lib/scoring.py` + 工具库 `lib/grading_utils.py`
- README 英文 + 中文双语，标题居中加粗

### 核心设计意图（内部，不公开）
- `.internal/DESIGN_INTENT.md` 存放设计意图，被 .gitignore 排除
- **公开代码保持中立**，但 Frontier 层设计针对顶级闭源模型的 6 大优势：
  1. 工作记忆容量 → X06（8 约束同时满足写故事）
  2. 元认知/自我纠错 → X07（数学题+验证+修正）
  3. 隐性知识挖掘 → X08（2 句 ticket 推 15 个隐藏需求）
  4. 归纳推理 → X09（从例子推断规则写代码）
  5. 安全判断力 → X10（同一请求里 3 做 2 拒）
  6. 深度分析推理 → X01-X05
- Leo 明确要求：**开源代码体现中立性，但核心要能测评选出最适合驱动 OpenClaw 的模型**
- 已清理所有提到 Opus/GPT-5.4/闭源优势的敏感词
