#### 技术教训
- GitHub README 渲染 release download URL 的 GIF 不可靠，大文件尤其如此。最好把 GIF 放 repo 内或用 `user-attachments` URL
- 子 agent 翻译大文件（>500行）容易超时——光读完就耗尽时间。解决方案：拆成更小的单文件任务，或直接手动处理
- ffmpeg 从 GIF 提取首帧做预览图质量很差（5.5KB），需要用 `-vframes 1 -q:v 2` 参数保证质量
- 子 agent 有时在超时前成功写入了文件但报 timed out——需要验证文件实际状态而不是只看 status
