## 一、产品定位

**Dit.ai 是全球AI算力流通协议。**

全球每年有数十亿美元的AI订阅额度被白白浪费——Anthropic Max、Kiro、Antigravity、OpenAI Pro的用户，大量额度每月清零。Dit让这些算力流动起来，成为可交易的商品。

Dit不是撮合平台，不是API代理。Dit是**AI算力的全球做市商**——供应商报底价，买家付顶价，平台掌握定价权，吃中间spread。

**v2.0新增定位：**
每一个OpenClaw实例天然是Dit的一个节点——既可以作为供应商贡献闲置算力，也可以作为买家以最优价格消费算力。目标：**年底前接入30万台OpenClaw实例**，构建全球最大的去中心化AI算力网络。

---

## 二、核心角色

### 供应商（Provider）
- 拥有AI订阅额度的OpenClaw实例（Anthropic Max、OpenAI Pro、Kiro、Antigravity等）
- 通过内置Dit Adapter自动接入网络
- 向Dit网络报价（每M token底价）
- 按实际成交量后结算收入

### 买家（Buyer）
- 需要AI算力的OpenClaw实例、开发者、企业
- 向Dit账户预充值
- 通过统一API接口调用（兼容OpenAI格式）
- 自动获得成本最优的算力路由

### Dit平台
- 维护Gateway路由层
- 掌握黑箱定价权（供应商不知道买家价格，买家不知道供应商底价）
- 管理预付款 + 后结算的资金流
- 不持有任何API key，不直接发起任何API请求

## 三、系统架构

```
OpenClaw实例（买家模式）
  │
  ▼
Dit Gateway（统一API入口）
  │  ├── 鉴权（实例API key）
  │  ├── 计费（扣买家余额）
  │  ├── 路由决策（全网最优供应商）
  │  └── 请求转发
  │
  ▼
OpenClaw实例（供应商模式，内置Dit Adapter）
  │  ├── 接收Gateway路由请求
  │  ├── 用自己的API key和IP发起请求
  │  └── 返回结果给Gateway
  │
  ▼
官方API（Anthropic / OpenAI / Kiro / Antigravity...）
```

**关键原则：**
- Dit平台永远不持有供应商的API key
- 所有对官方的请求，从供应商自己的IP发出
- Dit只是路由层，法律风险隔离清晰
- **每个OpenClaw实例可同时是供应商和买家**

## 四、定价机制

### 黑箱做市商模型

```
供应商报价（底价）  →  Dit平台掌握
买家定价（顶价）    →  Dit平台掌握
成交价              →  由Dit平台决定
Spread（差价）      →  Dit平台利润
```

**示例（Claude Opus 4.6）：**
| 角色 | 价格（per M input tokens） |
|------|--------------------------|
| 官方售价 | $15 |
| 供应商报价 | $9 |
| Dit对买家定价 | $12 |
| Dit毛利 | $3（25%毛利率） |

### 定价策略
- 对买家：始终低于官方价格10-20%（竞争力）
- 对供应商：高于其边际成本（吸引力）
- 中间spread随规模扩大而增大（规模效应）

## 五、资金流

```
买家预充值 → Dit账户（预付款，立即到账）
     ↓
Dit Gateway计费（按调用实时扣减）
     ↓
供应商月结（后结算，T+7或T+30）
```

**财务优势：**
- 买家预付 → Dit永远是净现金正向
- 供应商后结算 → 浮存金在Dit手中，随规模线性增长
- 零库存风险（算力即时消耗，不可囤积）

## 六、API设计

### 买家侧（兼容OpenAI格式）

```bash
# 买家只需修改base_url，其余代码零改动
curl https://api.dit.ai/v1/chat/completions \
  -H "Authorization: Bearer {DIT_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-opus-4.6",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

### 供应商侧（OpenClaw内置Dit Adapter）

```yaml
# OpenClaw实例配置：一键接入Dit网络
dit:
  enabled: true
  provider_mode: true          # 作为供应商贡献算力
  buyer_mode: true             # 作为买家消费算力
  models:
    - id: "claude-opus-4.6"
      api_key: "${ANTHROPIC_API_KEY}"
      price_per_m_input: 9.0   # 自动报价或手动设置
  gateway: "https://gateway.dit.ai"
```

## 七、路由决策逻辑

Gateway在收到买家请求后，按以下优先级选择供应商：

```
1. 价格最优（全网最低底价）
2. 延迟最低（地理位置优先，就近路由）
3. 稳定性（历史成功率 > 99.5%）
4. 负载均衡（单供应商不超过总流量5%，30万节点下天然分散）
```

故障转移：供应商响应超时（>3s）自动切换备用节点，买家无感知。30万节点规模下，故障转移速度 <100ms。

- **中性情景（$15/节点）：** 全年GMV ≈ $25M，年毛利 ≈ $5M

| 时间 | 月GMV | 平均浮存天数 | 浮存金余额 |

## 十、冷启动策略（修订版）

### 阶段一：种子节点（2026-Q1，0→1,000节点）
- MyClaw现有3,000用户，推送"接入Dit赚钱"入口
- 目标转化率30%，首批1,000节点
- 产品：MyClaw Dashboard一键接入，5分钟完成配置

### 阶段二：飞轮启动（2026-Q2，1,000→12,000节点）
- 供应商收入数据公开（脱敏）→ 激励更多人接入
- MyClaw新用户注册流程内嵌Dit接入引导
- 开发者文档+开源Adapter → 外部实例接入

### 阶段三：规模爆发（2026-Q3~Q4，12,000→300,000节点）
- Dit接入成为MyClaw实例的默认功能（opt-out而非opt-in）
- 企业级节点批量接入（API管理平台、AI SaaS）
- OpenClaw生态外的独立部署节点开放接入

### 关键激励设计
```
节点收入 = 实际交易额 × 供应商分成比例
          （Dit黑箱定价后，供应商拿到底价，Dit吃spread）

额外激励：
- 早期接入节点：前6个月额外+5%分成
- 稳定性奖励：30天在线率>99%，月末额外奖励
- 推荐奖励：推荐新节点接入，获得其首月交易额的3%
```

| 护城河 | 当前 | 30万节点时 |
|--------|------|-----------|

| **MyClaw绑定** | 3,000用户 | 50万+用户深度绑定，离开=失去收入 |
