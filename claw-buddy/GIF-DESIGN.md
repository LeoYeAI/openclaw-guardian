# Buddy-Claw GIF 动效生产机制设计

## 一、渲染引擎

### 技术路线（已验证）
- **位图绘制**：Node.js raw buffer 逐像素绘制（不用 SVG 字体）
- **5×7 位图字体**：每个字符映射为 5×7 像素点阵
- **像素放大**：每个逻辑像素 = 6×6 实际像素 → 输出约 480×530
- **GIF 编码**：sharp 输出 PNG 帧 → ffmpeg palettegen + paletteuse（dither=none）
- **结果**：像素锐利、文件小（~10KB）、Telegram 显示清晰

### 布局结构
```
┌─────────────────────────────┐
│        🍄 SPORE             │  ← 标题区：名字 + emoji
│    Common · Mushroom        │  ← 副标题：稀有度 + 物种
├─────────────────────────────┤
│                             │
│      .-o-OO-o-.             │  ← 精灵图区：3帧循环
│     (__________)            │
│        |×  ×|               │
│        |____|               │
│                             │
├─────────────────────────────┤
│   Lv.3 ⭐ | ❤️ 42          │  ← 状态栏：等级 + 好感度
│   buddy-claw · myclaw.ai    │  ← 品牌 footer
└─────────────────────────────┘
```

## 二、GIF 生成触发场景

### 场景矩阵

| 触发事件 | GIF 类型 | 帧数 | 特效 | 频率 |
|---------|---------|------|------|------|
| `/buddy gif` | 待机动画 | 3 | 物种 idle 动效 | 主动调用 |
| `/buddy` (status) | 状态卡 | 3 | idle + 状态信息叠加 | 主动调用 |
| 首次孵化 | 孵化动画 | 6 | 蛋→裂纹→破壳→宠物 | 一次性 |
| `/buddy pet` | 被摸反应 | 4 | 爱心飘起 ❤️↑ | 主动调用 |
| `/buddy feed` | 进食反应 | 4 | 食物→嚼→满足表情 | 主动调用 |
| 升级 | 升级庆祝 | 5 | 闪光 + "LV UP!" | 自动 |
| 稀有度展示 | 稀有光效 | 3 | 按稀有度不同光效 | `/buddy gif` |

### 不生成 GIF 的场景
- afterTurn 对话反应 → 纯文字（频率高，GIF 太吵）
- mute/name 操作 → 纯文字回执
- help → 纯文字

## 三、18 物种通用渲染规则

### 精灵图规范（继承自 Claude Code）
- 每个物种 **3 帧** idle 动画（已有数据）
- 每帧 **5 行 × 12 字符**
- `{E}` 占位符 → 替换为眼睛字符（·/✦/×/◉/@/°）
- 帽子覆盖第一行（如果不为 none）

### 位图字体覆盖表
需要映射的特殊字符：

| 字符 | 出现场景 | 像素设计 |
|------|---------|---------|
| `·` | 眼睛 | 中心1点 |
| `✦` | 眼睛 | 菱形4点 |
| `×` | 眼睛 | X交叉 |
| `◉` | 眼睛 | 圆环中心点 |
| `@` | 眼睛/蜗牛壳 | 螺旋 |
| `°` | 眼睛 | 小圆 |
| `ω` | 猫嘴 | 波浪 |
| `~` | 多种尾巴/水波 | 波浪线 |
| `^` | 龙/企鹅 | 尖角 |
| `/` `\` | 多种 | 斜线 |
| `=` | 猫/兔 | 双横线 |
| `<` `>` | 鸭/鹅喙 | 箭头 |
| `"` | 兔脚 | 两点 |
| `[` `]` | 乌龟壳/机器人 | 方括号 |
| `{` `}` | 六角恐龙腮 | 花括号 |
| `n` `u` | 水豚/仙人掌耳 | 小弧形 |

## 四、特殊动效帧设计

### 孵化动画（6帧）
```
帧1:  🥚 静态蛋
帧2:  🥚 蛋上出现裂纹（crack 字符）
帧3:  蛋裂开更大
帧4:  半壳 + 露出眼睛
帧5:  壳碎片 + 完整宠物
帧6:  宠物 + 名字 + 星星特效
```

### 被摸动画（4帧）
```
帧1:  正常 idle 帧0
帧2:  眼睛变成 ♥ + 上方出现 ❤
帧3:  眼睛变成 ♥ + 上方 ❤ 升高 + 新 ❤ 出现
帧4:  恢复正常 + 三个 ❤ 飘散
```

### 进食动画（4帧）
```
帧1:  食物 emoji 出现在嘴边
帧2:  嘴巴张开（替换嘴部字符）
帧3:  食物消失 + 嘴闭合 + 冒出 ✨
帧4:  满足表情（眼睛变 ◡）
```

### 升级动画（5帧）
```
帧1:  正常宠物 + XP 条满
帧2:  全身闪白
帧3:  恢复 + "LV UP!" 文字出现
帧4:  新等级数字放大
帧5:  正常 + 星星散落
```

## 五、生产流水线

### 渲染流程
```
输入: companion (species, eye, hat, rarity, shiny, growth)
  ↓
1. 查 BODIES[species] 获取 ASCII 帧数据
2. 替换 {E} → eye 字符
3. 覆盖 hat 到第一行
4. 根据触发场景选择帧序列（idle/hatch/pet/feed/levelup）
5. 每帧：
   a. 创建 raw RGB buffer (W × H × 3)
   b. 填充背景色（按稀有度）
   c. 逐字符查位图字体表 → 写入像素
   d. 叠加标题/状态栏/特效像素
   e. sharp: raw → PNG
6. ffmpeg: PNG 序列 → GIF (palettegen + paletteuse, dither=none)
7. 返回 GIF Buffer
```

### 文件结构
```
buddy-claw/
  src/
    gif-engine.ts      ← 核心渲染引擎（位图字体 + 像素绘制）
    gif-scenes.ts      ← 场景编排（idle/hatch/pet/feed/levelup 帧序列）
    bitmap-font.ts     ← 完整位图字体数据（ASCII + 特殊字符）
    gif-colors.ts      ← 稀有度配色方案
```

### 性能预算
| 指标 | 目标 |
|------|------|
| 单帧渲染 | < 50ms |
| 3帧 GIF 总耗时 | < 500ms |
| 6帧孵化 GIF | < 1s |
| 文件大小 | < 50KB (3帧) / < 100KB (6帧) |

## 六、集成到 buddy 命令

```
/buddy         → 文字状态卡 + idle GIF 附图
/buddy gif     → 纯 idle GIF
/buddy pet     → 文字 + pet GIF
/buddy feed    → 文字 + feed GIF
首次孵化        → 文字 + hatch GIF
升级            → 文字 + levelup GIF
```

GIF 作为 Telegram 动图附件发送（`message` action=send, filePath=生成的 gif）。

## 七、扩展性

- **新物种**：只需在 BODIES 加 3 帧 ASCII + bitmap-font 补充缺失字符
- **新场景**：在 gif-scenes.ts 加帧序列编排函数
- **贴纸包**：GIF 引擎输出 WebP 帧 → Telegram Sticker 格式
- **主题**：gif-colors.ts 可扩展为主题系统（赛博朋克/自然/像素RPG等）
