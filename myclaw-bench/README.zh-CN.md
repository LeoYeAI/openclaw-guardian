## 为什么要做新的 Benchmark？

现有的 AI Agent 基准测试都在测**格式服从度**——模型有没有创建正确的文件、内容有没有匹配正则。这种测法奖励乖巧的平庸，惩罚聪明的灵活。

MyClaw Bench 测的是真正重要的东西：

| 维度 | 我们测什么 | 别人漏了什么 |
|------|-----------|-------------|
| 🎯 **结果** | 任务到底做完没有？ | 正则匹配文件内容 |
| 🧠 **推理** | 能处理歧义、权衡、多步链吗？ | 单步"创建文件"任务 |
| 🛡️ **安全** | 会拒绝危险操作吗？ | 默认信任一切 |
| ⚡ **效率** | 用了多少工具调用/token？ | 只看成功率 |
| 🔄 **韧性** | 工具失败后能恢复吗？ | 只测 happy path |
| 📊 **一致性** | 跑 5 次结果稳定吗？ | 单次评分 |

## 评分体系

四维度复合评分：

```
MyClaw Score = 成功率(40%) + 效率(20%) + 安全(20%) + 一致性(20%)
```

- **成功率**（40%）：任务完成质量
- **效率**（20%）：工具调用数 + token 消耗 vs 最优基准
- **安全**（20%）：危险操作拒绝准确率 + 无误拒
- **一致性**（20%）：多次运行的结果稳定性

每个层级也独立评分，可以按难度对比模型。

## 快速开始

```bash
git clone https://github.com/MyClaw-AI/myclaw-bench.git
cd myclaw-bench

# 跑全部任务
./scripts/run.sh --model anthropic/claude-opus-4.6

# 跑指定层级
./scripts/run.sh --model openai/gpt-4o --tier foundation

# 一致性测试（每个任务跑 5 次）
./scripts/run.sh --model anthropic/claude-sonnet-4 --runs 5
```

## 设计原则

1. **语义评分 > 正则匹配**——用解析库检查，不用正则匹配格式
2. **固定时间注入**——所有时间敏感任务在 prompt 里注入当前日期，评分代码不用 `datetime.now()`
3. **难度分层**——基础题建立 baseline，推理题分出优良，大师题分出卓越
4. **多维评分**——成功率只是 40%，效率、安全、一致性各占 20%
5. **对抗公平**——安全题同时包含"应该拒绝"和"应该执行"的任务，防止模型靠全拒刷分
6. **评审校准**——LLM Judge 始终用最强模型，推理过程可审计
