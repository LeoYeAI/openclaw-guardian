# MyClaw 生态增长战略 v2.0

_目标：建立多样化的生态节点，每天稳定产生新增实例_
_Last updated: 2026-03-21 | Author: The Doctor 🌀_

---

## 🧭 核心逻辑

MyClaw 不是应用，是平台。多条并行获客路径汇入同一套基础设施，驱动实例增长。

```
【流量入口层】（并行，互不依赖）
  开源生态       Influencer联盟    Install按钮    第三方推荐站    硬件平台
（开发者流量）  （KOL/训练营/IT顾问）（Skill/App生态）（SEO长尾）   （设备激活）
       ↓              ↓               ↓              ↓            ↓
【基础设施层】（MyClaw 唯一要做好的技术产品）
              开发者 API + MyClaw 实例（订阅 + Token）
                            ↓
【增长目标】
         每日新增实例数 ↑        Token 消耗 ↑
```

**核心原则：**
MyClaw 只做两件事——**把 API 做到极致好用**，**把各入口的激励机制设计好**。
具体场景、应用、内容，全部交给生态方。

## 🌐 第零层：开源生态（流量总闸）

这是所有节点的上游，做好了其他一切都受益。

### 动作 1｜成为 OpenClaw 官方推荐托管方

> ⚠️ **落地姿势注意**：OpenClaw maintainer 有商业意识但极度在意社区观感。不能直接要求"在 README 里推荐 MyClaw"——这会被其他托管商和社区贡献者施压反弹。正确姿势是：技术贡献建立信任 + 生态基金资助社区，推荐是水到渠成的结果，不是谈判条件。

两件事，同步做：

**① 派工程师提 PR**
让 1-2 个工程师专门负责给 OpenClaw 贡献代码——修 bug、提改进、参与讨论。不谈商业，只谈贡献。3 个月后你在社区有了技术信任，这是买不到的东西，也是竞争对手无法复制的壁垒。

**② 设立 OpenClaw 生态基金**
每个通过 MyClaw 托管的付费用户，捐出固定金额（如 $1-2）进入生态基金，专项资助 OpenClaw 核心开发者和社区活动。这不是一次性买断，是持续的利益绑定——MyClaw 越大，基金越大，OpenClaw 官方越有动力持续推荐你。基金专项用于：核心维护者的开发补贴、社区 Hackathon、OpenClaw 官方文档改进。

**目标（不是谈判要求，是自然结果）：**
- OpenClaw README 加「推荐托管：MyClaw」官方链接
- OpenClaw 安装流程内嵌「一键部署到 MyClaw」（这个才值大钱）
- **价值：OpenClaw 27.5 万 stars，每天数万精准高意向 UV，永久免费流量**

### 动作 2｜开源工具矩阵 + 生态开发者项目

两层结构：MyClaw 官方工具覆盖开发者工作流；生态开发者的开源项目和 Agent Apps 通过安装量和使用量带动实例增长——包括**非 OpenClaw 用户首次通过 App 激活 MyClaw 实例**，App 本身即获客入口。

**第一层：MyClaw 官方开源工具**
每个工具 README 带「Powered by MyClaw」+ 注册链接，持续品牌曝光：

| 工具 | 作用 |
|------|------|
| myclaw-sdk-js / python | API 集成必用，开发者天天接触 |
| myclaw-cli | 命令行管理实例 |
| myclaw-templates | 常用 Agent 模板库，fork 即用 |
| openclaw-installer | 一键装 OpenClaw，结尾推 MyClaw 托管 |

**第二层：生态开发者开源项目 + Agent Apps**
MyClaw 提供 API + 分成激励，开发者自主构建，安装量和使用量反哺实例增长：

| 类型 | 逻辑 | 例子 |
|------|------|------|
| 开源集成项目 | 发布到 GitHub，带 Install 按钮，安装量带来新实例 | MyClaw × Notion 集成、MyClaw × Slack Bot、自定义 MCP server |
| Agent Apps | 非 OpenClaw 用户使用 App → 引导创建 MyClaw 实例 → 首次激活；使用消耗 Token → 带动收入 | 任务自动化助手、多 Agent 协作平台、垂直场景助手（法律/财务/教育）|

**飞轮：** 工具 / App 使用量 → 安装量 → 新实例创建 → Token 消耗 → 开发者分成 → 开发者主动推广 → 更多安装量

### 动作 3｜「Install on MyClaw」按钮——Skill/App 联合增长引擎

这是一个独立的增长引擎，逻辑类比 Vercel 的「Deploy to Vercel」按钮，但更强：**它同时解决了用户获取和 skill 开发者商业化两个问题**。

**三类核心对象：**

| 类型 | 具体例子 | 为什么会放按钮 |
|------|----------|----------------|
| **OpenClaw Skills** | 各类免费 skill（爬虫/代码/助理等）| 直接商业化，被动收入 |
| **可部署到 OpenClaw 的开源服务** | n8n、Flowise、Open-WebUI、SearXNG、Browserless、Firecrawl、Langfuse、Dify 等 | 获得更多用户，扩大安装量 |
| **面向 AI Agent 的产品/工具** | MCP server、Agent框架、评测工具、向量DB、Tool套件 | 用户即目标客户，精准导流 |

这三类覆盖了 OpenClaw 生态的完整上游——skill 是最细粒度的扩展，开源服务是基础设施层，Agent 工具是平台层。任何一类开发者放了按钮，都在向 MyClaw 输送精准用户。

**核心机制：**
- MyClaw 提供一个标准化嵌入按钮（SVG badge + 跳转链接），以上三类开发者都可以放在：
  - GitHub README（一行 Markdown）
  - 官网首页 Hero 区域
  - 文档站、ProductHunt 页面
- 用户点击后进入 MyClaw 托管的快捷安装流程：
  1. **已有 OpenClaw 实例**：一键安装该 skill/服务到自己的实例
  2. **没有实例**：实时创建一台新 OpenClaw 实例 + 预装目标服务，全程 < 60 秒

**开发者/项目方商业化激励（这才是让对方主动铺按钮的原因）：**

| 触发行为 | 获得 | 适用对象 |
|----------|------|----------|
| 用户通过按钮创建新实例并订阅 | 首月订阅收入 20% | 三类全部 |
| 用户通过按钮安装并消耗 Token | Token 消耗持续 8% | Skills / Agent 工具 |
| 开源服务项目方（非商业）| 用户量统计 + 官方 Featured 位 | 开源服务 |
| Verified 认证额外加成 | 分成比例 +3% | Skills |

**飞轮效应：**
```
免费 skill 开发者放按钮（零成本）
        ↓
用户点击 → 进入 MyClaw 流程 → 新实例创建
        ↓
开发者获得持续 Token 分成
        ↓
开发者主动宣传自己的 skill → 带来更多用户
        ↓
MyClaw 获得精准高意向用户（已知需求场景）
```

**与「Deploy to Vercel」的核心差异：**
Vercel 按钮只解决部署，开发者无收益；MyClaw 按钮让开发者有**持续被动收入**，驱动力完全不同。免费 skill 变成了开发者的「分销渠道」。

**执行路径：**
- 技术：1个接口 + 1个落地页，2周 MVP
- **冷启动前置动作（按钮上线前必须完成）**：制定开发者触达专项方案——通过 OpenClaw Discord、GitHub Contributors 列表、ClawHub 作者页找到 top 50 skill 作者的联系方式，建立直接沟通渠道
- 冷启动优先级：
  1. **ClawHub top 50 热门 skill 作者**（最快落地，直接给 Verified + 分成加成）
  2. **高星 MCP server 项目**（GitHub 上 MCP server 生态已有上千个，精准对口 OpenClaw 用户）
  3. **n8n / Flowise / Firecrawl 等开源服务**（装机量大，作者有动力铺按钮换用户数据）
- 传播：每个合作方的 README 都变成 MyClaw 的免费广告位，覆盖范围远超付费渠道

### 动作 4｜与第三方 Skill 推荐站合作，嵌入「Deploy on MyClaw」按钮

> ⚠️ **待验证入口**：大厂（如 Google、GitHub）有相关聚合页，但专注 OpenClaw skill 的独立小站尚未经过系统调研。**执行前提：先做市场调研，确认这类站实际存在且有流量，再投入合作资源。**

生态里存在大量靠 SEO 吃流量的独立站——收录 OpenClaw skill、做测评和推荐，用户搜索 skill 时自然落地。这类站本身缺乏变现手段，MyClaw 的按钮对他们是增值，对 MyClaw 是精准的 skill 安装流量入口。

**合作逻辑：**
- 站方在每个 skill 详情页放「Deploy on MyClaw」按钮
- 用户点击 → 已有实例直接安装；没有实例 → 引导注册 + 预装 skill，全程 <60 秒
- MyClaw 给站方流量分成或固定合作费，激励其主动推广

**执行路径：**
1. 找到生态里已有的 skill 推荐站（Google 搜索 + OpenClaw 社区）
2. 主动联系合作，提供标准按钮 embed 代码 + 分成方案
3. 数据跑通后，作为标准合作模板批量复制

### 动作 5｜AI 硬件生态平台

MyClaw 不做硬件，但做硬件的**精准投流代理商 + 开发基础设施 + 分发平台**三合一。官网开设硬件板块，所有入驻硬件默认接入 MyClaw 云端能力。

> **核心筹码重新定义**：硬件平台的门槛不是用户规模，而是 MyClaw 的**投流能力**。MyClaw 可以为每一款硬件单品单独投流，厂商按实际激活效果付费——这是传统硬件厂商无法独立实现的精准获客能力。体量不是谈判前提，投流 ROI 才是。

#### 平台定位：硬件创业闭环

```
开发阶段  →  用 MyClaw 实例镜像直接开发，无需自建后端
测试阶段  →  在 MyClaw 用户池里内测，真实反馈最快
上市阶段  →  上架 MyClaw 硬件平台 + MyClaw 接管投放带货
增长阶段  →  订阅费 + Token 持续生态分成
```

对硬件创业团队：MyClaw 是他们的 **AWS + App Store + 广告代理商** all-in-one。

#### 收入结构

| 来源 | 说明 |
|------|------|
| **投放预算** | 厂商给 MyClaw 广告预算，MyClaw 用投放能力跑 ROI，超出部分是利润 |
| **生态分成** | 设备激活后的订阅费 + Token 消耗，按生态合作标准分成（双向激励） |

#### 硬件接入方式（双轨并行）

**后置接入**（冷启动，快速铺量）
- 用户买到硬件后，扫码 / 输激活码绑定 MyClaw，5 分钟完成
- 门槛极低，无需厂商改固件，任何品类都能快速谈

**预装接入**（深度合作，丝滑体验）
- 设备出厂默认指向 MyClaw，开机即激活
- 适合已跑通 ROI 数据、出货量大的战略合作厂商

#### 选品范围

覆盖全品类——消费端（AI 音箱、AI 眼镜、AI 伴侣机器人、AI Pin）+ 极客/企业端（边缘计算盒子、开发板）。选品核心标准：能接 OpenClaw + 厂商愿意给投放预算 + 用户 LTV 健康。

#### 执行路径

1. **开发者招募**：面向早期硬件团队，提供 MyClaw 实例镜像 + API，支持从第一行代码开始在 MyClaw 上开发
2. **首批入驻**：筛选 3–5 款已有 OpenClaw 社区集成的硬件，用后置接入方式快速上线
3. **跑通 ROI**：第一个品类验证投放模型，形成可复制的标准合作模板
4. **开放平台**：建立「MyClaw 硬件认证」体系，开放厂商自助申请入驻，平台化运营

### 动作 6｜Influencer & Partner 联盟

同一套增长方法：用**长期分成合同**把外部推广者变成永久合伙人，按触达对象分三种角色。

| 角色 | 触达对象 | 合作模式 | 分成 |
|------|----------|----------|------|
| 内容 KOL | 个人用户 | 纳米 KOL 批量（500 人 × 50 注册）<br>一次性付费 → 长期分成合同 | 纳米：30%首月 + 8%持续<br>中腰：25%首月 + 6%持续 |
| AI 训练营 | 学习者 | 批发实例 $15/月打包进课程，学员天然留存 | 7折批发价差 |
| IT 顾问 / MSP | 中小企业 | 顾问收客户 $200–500/月，MyClaw 实例 $39，毛利厚 | 持续订阅流水 |

**核心逻辑**：YouTube 视频永久产生流量 = 永久被动收入，KOL 主动接受；训练营和顾问赚差价，自驱推广。

## ⚙️ 核心基础设施：开发者 API Platform

**这是整个生态战略的地基。入口一（嵌入场景）不再由 MyClaw 自己做，全部开放给开发者，他们用 API 构建。**

### MVP 五个核心接口

```
POST   /v1/instances              创建实例
GET    /v1/instances/:id          查询状态
DELETE /v1/instances/:id          销毁实例
POST   /v1/instances/:id/message  向实例发消息
POST   /v1/instances/:id/topup    充值 token
```

加 Webhook：实例状态变更 / token 余额不足时回调通知开发者。

### 开发者体验（DX）要求
- 5 分钟快速开始，复制 3 行代码跑通第一个实例
- Sandbox 环境（测试不扣费）
- Dashboard：实时查看带来的实例数 + 累计收益

### 开发者分成（激励生态构建者）

| 月带量 | 订阅分成（首月）| Token 分成（持续）|
|--------|----------------|------------------|
| 1–9 个 | 15% | 无 |
| 10–99 个 | 20% | 5% |
| 100–499 个 | 30% | 8% |
| 500+ | 定制 | 定制 |

**激励叙事核心：永久被动 Token 收益，不是首月分成。**
首月分成是一次性的；Token 持续分成才是真正的钩子——开发者做好一次产品，用户越活跃、开发者赚越多，天然绑定，无需主动运营。向开发者沟通时，重点呈现"月均活跃用户 × Token 消耗 × 分成比例"的长期复利模型，而非首月百分比。

### Partner Marketplace
在 MyClaw 官网做「Built on MyClaw」展示页：
- 所有基于 API 构建的应用上架展示
- 用户在这里发现各类垂直应用（Shopify 版、Notion 版、教育版…）
- 开发者有展示位和流量激励，持续做高质量产品
- MyClaw 不生产内容，只运营市场

2. dev.myclaw.ai 开发者文档独立站
3. HackerNews Show HN 发布
4. Product Hunt Developer Platform 发布

## 🎯 战略本质

**v1.0 的思路：** MyClaw 自己铺多个入口

**v2.0 的收敛：** MyClaw 做平台，生态方铺入口

> MyClaw 的角色是平台运营者，不是应用开发者。
> 把 API 做好，把激励设计好，让一千个开发者替你创造一千个入口。
> 这才是 Stripe 真正的秘密。

---

_v2.0 更新说明：入口一（嵌入型）不再由 MyClaw 自建，合并入开发者 API 生态，由开发者用 API 自主构建各类垂直场景。MyClaw 专注平台基础设施和激励机制。_
